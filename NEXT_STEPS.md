# 次のステップ（引き継ぎチェックリスト）

このドキュメントでは、現行版の TERASS 業務サポート AI を引き継ぐ際に実施すべきタスクをまとめています。チェックリスト形式で記載しているので、作業の進捗管理にお役立てください。

## ✅ 当面の優先タスク（P0）

1. **リリースパッケージのコミット**
   - `terass_full_package_updated.zip`、`RELEASE_NOTES.md`、`NEXT_STEPS.md` をリポジトリに追加し、PR を作成して main ブランチへマージします。
   - リポジトリに直接書き込む場合は `git add`→`git commit`→`git push` を実行し、保護ルールに従って PR を経由してください。

2. **iOS 設定ファイルの更新**
   - `terass_ios_app/src/config.js` 内の `VECTOR_API_URL` および `KNOWLEDGE_API_URL` を本番環境のエンドポイントに置き換えます。
   - Expo/EAS で再ビルドし、TestFlight で正しく接続できることを確認します。

3. **Bitwarden シークレット名の最終統一**
   - 既存のシークレットで日本語名が残っている場合は `bws secret edit --key <新しいキー名> <SECRET_ID>` で英数字＋アンダースコアへリネームします。
   - プロジェクトIDごとにシークレットを整理し、UUID をキー名に使う場合は `--uuids-as-keynames` オプションを設定します。

## 🗓️ 今週中に進めるタスク（P1）

- **自動化キットの本番運用**
  - GitHub Actions またはサーバ上の Cron で `terass-automation-kit` を定期実行し、ナレッジベースの更新とベクタDB の再インジェクションを自動化します。
  - `.env.example` を `.env` にコピーし、Bitwarden Secrets Manager から必要な API キーや認証情報を注入する仕組みを整えます。

- **FAQ/入力補助の拡充**
  - phase2_docs に記載されているアイデアをレビューし、よくある質問の追加やフォーム入力サポート機能の仕様を決定します。

- **ユーザーテスト (2 回目) の準備**
  - 1 回目のテスト結果に基づいてアプリを修正後、新しいテスターを募集し、同じシナリオで再度検証します。
  - 検証用のシナリオと評価項目を整理し、フィードバックを収集する仕組みを整えます。

## 🔭 長期的なタスク（P2 以降）

- **ローン診断機能**：ユーザー条件に応じたローン提案アルゴリズムを設計し、UI を用意します。
- **TERASS Picks 連携**：物件情報をリアルタイムで取得する API を検討し、チャット内で参照できるよう統合します。
- **UI/UX の継続的改善**：ユーザーテスト 3 回目を実施し、残っている問題点や新たな要望を取り込んで改善を続けます。
- **自動化キットの機能拡張**：RAG に利用する資料の追加収集先（Notion、Google Drive など）を広げ、ベクタDB 更新を含む運用 Runbook を作成します。

## 🧪 最低限の品質確認チェック

- `OPENAI_API_KEY` 未設定時にエラーが表示され、Bitwarden での設定方法を案内しているか。
- ネットワーク断時に適切なエラーメッセージと再試行方法が提示されるか。
- Secrets 未設定 (`bws run` を使わずに起動) の場合に、何が足りないか分かりやすく表示されるか。
- phase2_docs や FAQ が欠落している場合にアプリが落ちないか、適切なフォールバックがあるか。

このチェックリストをもとに引き継ぎを進めれば、現在の実装と今後の拡張の両方をスムーズに管理できます。必要に応じてタスクを細分化し、担当者と期限を設定して運用してください。
